# Security Policy

## Supported Versions

Currently, only the latest iteration of the `ystack` `master/main` and `dev` branches are supported for security updates.

## Reporting a Vulnerability

If you've found a vulnerability, thank you for helping us keep this open-source tool safe!

Because `ystack` uses AI models and tools (like LLM invocation, IDE automation, shell script execution), vulnerabilities can directly affect developers' local systems.

**Please DO NOT open a public issue.** Instead, responsibly disclose the vulnerability by sending an email directly to the maintainer or opening a private GitHub Security Advisory. 

We take all vulnerabilities seriously and will work with you to analyze, fix, and credit the findings according to standard open-source responsible disclosure timelines.
