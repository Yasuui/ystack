# ystack Bootstrap Prompt
#
# Use this when: starting ystack on a new project for the first time.
# Where: Antigravity Agent Manager (Cmd/Ctrl+Shift+M) OR Gemini CLI chat.
# Result: fully configured AGENTS.md + GEMINI.md + project health baseline in ~2 minutes.
# Only ever run this ONCE per project. Use /context to update after that.
# ─────────────────────────────────────────────────────────────────────────────

You are bootstrapping the ystack swarm system for this project. 
Run everything autonomously. Do not ask for confirmation.

## Step 1 — Read the codebase (before writing anything)

Scan the project:
```bash
find . -maxdepth 3 \
  -not -path "*/node_modules/*" \
  -not -path "*/.git/*" \
  -not -path "*/.next/*" \
  -not -path "*/dist/*" \
  -not -path "*/__pycache__/*" \
  -not -path "*/target/*" \
  2>/dev/null | head -80
```

Then read these files if they exist:
- `package.json` — framework, scripts, dependencies
- `requirements.txt` or `pyproject.toml` — Python deps
- `Cargo.toml` — Rust/Tauri
- `src-tauri/tauri.conf.json` — Tauri config
- `next.config.js` / `next.config.ts` — Next.js config
- `.env.example` or `.env.local` — integration hints (keys only, never values)
- Any existing `README.md` — project description

## Step 2 — Create .agents/ structure

```bash
mkdir -p .agents/brain .agents/skills
touch .agents/brain/.gitkeep
```

Add to `.gitignore` if not present:
```
.agents/brain/*
!.agents/brain/.gitkeep
.agents/skills/taste-soft-skill/
```

## Step 3 — Install taste-soft-skill

```bash
git clone https://github.com/Leonxlnx/taste-skill.git /tmp/taste \
  && cp -r /tmp/taste/soft-skill .agents/skills/taste-soft-skill \
  && rm -rf /tmp/taste
```

## Step 4 — Copy swarmstack agent prompts

If the `swarmstack/` folder doesn't exist in this project, clone ystack and copy:
```bash
git clone --depth=1 https://github.com/younis-ali/ystack /tmp/ystack \
  && cp -r /tmp/ystack/swarmstack . \
  && rm -rf /tmp/ystack
```

```markdown

## What this is

## Stack
[Same as AGENTS.md stack section]

## Architecture
[Describe what you see: key directories, how data flows, main entry points]

## How to run

## Code conventions
[Inferred from existing code patterns]

AGENTS.md is active. Swarm agents in swarmstack/agents/.
Use swarmstack/orchestrator.md in Agent Manager for feature work.
```

## Step 7 — Establish health baseline

Run the test suite:
```bash
# JavaScript/TypeScript
npm test 2>/dev/null || npx vitest run 2>/dev/null || echo "No test runner found"

# Python  
pytest --tb=no -q 2>/dev/null || echo "No pytest found"
```

Write `project-health-state.md`:
```markdown
# Project Health State — [project name]
Updated: [today's date]

## Status: 🟡 Needs Attention (baseline run)

Test coverage:  [result from test run, or "not measured yet"]
Failing tests:  [count from test run, or "not run"]
Security audit: not run
Last swarm run: [today's date]

## Next session priorities
1. Establish test coverage baseline
2. Run security audit (swarmstack/agents/review.md)
3. [Any issues detected during bootstrap scan]
```
